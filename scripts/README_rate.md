# אימות מקור ריבית בנק ישראל (rate.json)

הפאנל במחשבון מציג את ריבית בנק ישראל והפריים, ומתעדכן אוטומטית
ע"י ה-GitHub Action שב-`.github/workflows/update-rate.yml`, שמריץ את
`scripts/update_rate.py`.

הסקריפט מושך את הריבית ממאגר הסדרות הרשמי של בנק ישראל
(`edge.boi.gov.il`, פורמט SDMX). **לפני ההפעלה הראשונה יש לאמת שה-URL
של הסדרה (`SERIES_URL` בסקריפט) באמת מחזיר את הריבית.**

## נוהל אימות (פעם אחת)

1. היכנס למאגר הסדרות: https://edge.boi.gov.il/?locale=he
   (תפריט: נתונים וסטטיסטיקה ← מאגר הסדרות).
2. חפש "ריבית בנק ישראל" / "BOI interest rate". פתח את הסדרה ורשום את
   ה-dataflow ואת קוד הסדרה (SERIES_CODE).
3. בנה URL בפורמט CSV עם התצפית האחרונה, בצורת מפתח-בנתיב (סינון
   לסדרה אחת — חשוב, כי הסקריפט לוקח את התצפית האחרונה בתשובה):
   `https://edge.boi.gov.il/FusionEdgeServer/sdmx/v2/data/dataflow/BOI.STATISTICS/<DATAFLOW>/1.0/<SERIES_CODE>/?lastNObservations=1&format=csv`
   (הסדרה הנוכחית: dataflow `BR`, סדרה `MNT_RIB_BOI_D` = ריבית בנק ישראל.)
4. בדוק שהסקריפט מצליח לפענח אותו:
   ```
   python scripts/update_rate.py "<ה-URL-שבנית>"
   ```
   הפלט צריך להראות ערך סביר (נכון ל-30/06/2026 — **3.75%**, פריים 5.25%).
5. כשה-URL נכון — עדכן את הקבוע `SERIES_URL` בראש `update_rate.py`,
   והרץ שוב בלי ארגומנט לאישור.

## הבטחת בטיחות

אם המשיכה נכשלת או מחזירה ערך לא-סביר, הסקריפט **אינו כותב דבר** —
`rate.json` נשאר עם הערך האחרון הידוע (כולל התאריך שלו). כך הפאנל
לעולם לא יציג מספר שגוי שמתחזה ל"חי".

## קצב עדכון

ריבית בנק ישראל משתנה כ-8 פעמים בשנה בלבד, במועדים ידועים מראש.
ה-Action רץ יומית, אבל ברוב הימים פשוט ימצא את אותו ערך ולא ישנה כלום.
